# Система мониторинга простого веб-приложения с автоперезапуском

Надёжная отказоустойчивая система, которая следит за работоспособностью небольшого Flask-приложения «Hello World» и автоматически перезапускает его при падении или недоступности.

### Возможности
- Проверка каждые N секунд (настраивается)
- Успешным считается **любой HTTP-код 2xx** + наличие строки `Hello World` в теле ответа
- Автоматический перезапуск приложения при сбое
- Корректное завершение всех дочерних процессов
- Полное логирование всех событий (в файл + консоль)
- Автозапуск мониторинга при старте системы (через systemd)
- Простая установка и обновление одной командой
- Все настройки вынесены в `config.yaml`

### Структура проекта
```
monitoring-system/
├── app/                    # Само веб-приложение
│   ├── app.py              # Flask "Hello World"
│   ├── venv/               # виртуальное окружение (создаётся при установке)
│   └── requirements.txt
├── monitor/                # Скрипт мониторинга
│   ├── monitor.py          # основной скрипт
│   ├── config.yaml         # конфиги
│   └── log/
│       └── monitor.log     # логи работы мониторинга
├── systemd/
│   └── web-monitor.service # юнит для автозапуска
├── install.sh              # установка и настройка системы
├── update.sh               # безопасное обновление проекта
├── start_monitoring.sh     # ручной запуск (для тестов)
└── README.md               # в пояснении не нуждается
```

### Быстрая установка (одной командой после клонирования)
```bash
git clone https://github.com/tvoy_nik/monitoring-system.git
cd monitoring-system
chmod +x install.sh
./install.sh
```

После выполнения скрипт:
- установит виртуальное окружение и зависимости
- создаст все нужные папки
- скопирует systemd-сервис
- включит автозапуск при загрузке системы

### Управление службой (systemd)
```bash
# Запуск мониторинга
sudo systemctl start web-monitor.service

# ОК

# Остановка
sudo systemctl stop web-monitor.service

# Перезапуск
sudo systemctl restart web-monitor.service

# Статус (очень полезно!)
sudo systemctl status web-monitor.service

# Посмотреть логи в реальном времени
journalctl -u web-monitor.service -f
```

### Ручной запуск (для тестирования и отладки)
```bash
./start_monitoring.sh
```

### Обновление проекта (после git pull)
```bash
./update.sh
```
Скрипт сам остановит мониторинг → обновит зависимости → запустит заново.

### Настройка (monitor/config.yaml)
```yaml
app_url: "http://127.0.0.1:5000"   # адрес вашего приложения
check_interval: 10                # проверка каждые 10 секунд
timeout: 5                        # таймаут запроса в секундах
```

Меняйте значения под свои нужды — перезапускать мониторинг не нужно, он перечитывает конфиг при каждой итерации.

### Где смотреть логи
- Основные логи мониторинга: `monitor/log/monitor.log`
- Логи systemd: `journalctl -u web-monitor.service -f`
- Логи самого Flask-приложения — в консоли или в systemd (если запускать через сервис)

### Тестирование работы
1. Запустите мониторинг
2. Убейте процесс приложения вручную:
   ```bash
   pkill -f app.py
   ```
3. Через пару секунд в логах увидите:
   ```
   WARNING  Приложение недоступно! Перезапуск...
   INFO     Приложение запущено с PID 12345
   INFO     Приложение работает корректно
   ```
